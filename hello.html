Recap GIT !

Les commits

git init
il fait du repertoire spécifié (par defaut le rep courant sinon) un depot git.
git status
Affiche les infos sur ce qui n'a pas encore été commit entre autres et d'autres infos.
git diff
Affiche les modif apportés à tous les fichiers si on ne precise aucun fichier à la suite. on peut ajouter l'option --cashed pour que la verification se fasse au niveau de la zone d'indexation.
git add
Ajoute les fichiers spécifiés dans la zone d'indexation. si rien est précisé il prend tous les fichiers de l'espace courant.
git commit -m "message"
Ajoute maintenant le contenu de la zone d'indexation en tant que commit.
git log
Affiche la liste des commits jusqu'à l'actuel (le HEAD)
git checkout
Prend en parametre le SHA-1 ou le Tag du commit vers lequel on veut naviguer. "Master" pour le dernier.
git tag
Affiche la liste des tags qu'on a créé.
git tag -nom-
Affecte le tag au commit courant.
git tag --delete -nom-
Supprime le tag.

Les dépots distants

git clone "remote link" "dir"
Clone le depot distant en local.
git remote -v
Affiche la liste des remotes (push pour uploader et fetch pour recuperer).
git remote show "nom-remote"
Affiche les infos détaillées sur le remote/dd>
git remote add "nom-remote" "lien-push"
Synchronise le depot distant au depot local.
git push -u "nom-remote" "nom-branche"
Envoie le contenu du depot local vers le depot distant lié.
git push origin "nom-tag"
Envoie le tag spécifié depuis le depot local vers le dépot distant.
git push (origin) --tags
Envoie tous les tags cette fois ci.
git fetch
Recupere les commits et autres depuis le serveur distant (remote). en local, le tag Master reste tjrs sur le commit vers lequel il pointait.
git pull
Applique ces modifications provenant du serveur à notre depot local. le Tag Master va alors pointer vers le dernier commit de meme que "origin/master"
